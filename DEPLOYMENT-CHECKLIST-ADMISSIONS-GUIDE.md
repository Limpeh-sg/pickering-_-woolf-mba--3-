## 🚀 部署步骤

### 1. 最终构建
```bash
npm run build
```

### 2. 检查构建产物
```bash
ls -lh dist/
```

预期文件：
- `index.html`
- `assets/index-*.js` (~870KB)
- `assets/index-*.css` (~117KB)
- `assets/favicon-new-*.png`
- `assets/og-image-new-*.png`

### 3. 测试生产构建
```bash
npm run preview
```

访问 http://localhost:4173 测试

### 4. 部署到服务器
根据您的部署方式：
- **Vercel**: `vercel --prod`
- **Netlify**: `netlify deploy --prod`
- **自托管**: 上传 `dist/` 目录

## 🎓 顾问培训

### 培训内容
1. **系统介绍**
   - 这是决策树系统，不是聊天机器人
   - 用户会经过5级选择
   - 每个线索都有分数和标签

2. **线索等级**
   - Hot Lead（≥70分）：30分钟内联系
   - Warm Lead（40-69分）：24小时内联系
   - Cold Lead（<40分）：进入nurture list

3. **Payload解读**
   - 如何查看用户选择路径
   - 如何理解标签含义
   - 如何使用顾问建议

4. **跟进话术**
   - 不要问"你想了解什么"（用户已经回答过）
   - 根据路径使用对应话术
   - 参考顾问建议执行动作

- [ ] 系统演示视频
- [ ] Payload示例文档
- [ ] 跟进话术手册

## 🐛 故障排查

### 问题1：助手按钮不显示
**检查**：
- z-index是否正确（应为60）
- 是否被其他元素遮挡
- 动画是否完成加载

### 问题2：选项点击无反应
**检查**：
- 控制台是否有错误
- 数据结构是否正确
- 事件处理是否绑定

### 问题3：表单提交失败
**检查**：
- API端点是否可访问
- 网络请求是否成功
- Payload格式是否正确
- CORS配置是否正确

### 问题4：进度条不更新
**检查**：
- state.currentLevel是否正确更新
- 动画是否正常执行
- CSS是否正确加载

### 问题5：中英文切换异常
**检查**：
- lang prop是否正确传递
- 所有文案是否都有双语版本
- 条件渲染是否正确
